## 1. Theme

M1 established the **end-to-end geo-sonification pipeline** and made the foundational design pivot that shaped every later milestone: from "20-year deforestation time series" to **"interactive map + real-time present-day data, streamed via OSC to Max/MSP."** Everything M2 built on (sound design, the seven-bus ambience, the eventual web-audio rewrite) starts from the architecture and OSC contract laid down here.

The pipeline at M1 close:

```
GEE export (CSV, 0.5° global grid) → Node.js server (viewport aggregation) → OSC :7400 → Max/MSP patch (max_wav_osc.maxpat)
```

Two operating modes, threshold-switched:

- **Aggregated** (gridCount > 50): one set of viewport-level stats + 11-class landcover distribution.
- **Per-grid** (gridCount ≤ 50): per-cell messages with viewport-relative position and per-cell landcover distribution, intended to feed a Max `poly~` voice allocator.

## 3. Architecture at M1 close

### 3.1 OSC contract

**Aggregated mode** (15 messages):
- `/landcover` (int, dominant ESA class)
- `/nightlight`, `/population`, `/forest` (float 0–1, normalized)
- `/lc/10` … `/lc/100` (11 floats, area fractions; classes not present send 0.0)

**Per-grid mode**, on every viewport update:
```
/mode        "per-grid"
/grid/count  N
/viewport    west south east north
  × N cells:
    /grid      lon lat landcover nightlight population forest
    /grid/pos  xNorm yNorm
    /grid/lc   f10 f20 f30 f40 f50 f60 f70 f80 f90 f95 f100
```
Aggregated OSC is also sent in per-grid mode — Max can choose freely.

### 3.2 Server modules introduced or shaped during M1

- `server/index.js` — top-level boot, CSV parse with `normalizeLandcoverClass()`, `processViewport()` shared helper, per-client mode state map (later extracted into `server/mode-manager.js` in commit `496d6a4`, a few days after the last M1 devlog entry; lived as the dedicated module through M2/M3 before being merged into `client-state.js` at M4 P4-3).
- `server/spatial.js` — viewport queries via 2D bucket index; returns `gridsInView`.
- `server/normalize.js` — extracted `normalizeOscValues()` for reuse across both modes.
- `server/osc.js` — `sendModeToMax()`, `sendToMax()` (aggregated), `sendGridsToMax()` (per-grid + position + lc distribution).
- `server/landcover.js` — `getCellLcDistribution()` helper (with fallback synthesis from discrete class).
- `server/config.js` — `PER_GRID_THRESHOLD` env var (default 50).

### 3.3 Per-client state

- WebSocket: state keyed by connection (one client = one connection).
- HTTP: state keyed by client IP, with **5-minute TTL** — first appearance predates M4's `client-state.js` by ~3 months; the TTL pattern lived through M2/M3 unchanged before being unified at M4 P4-3.

## 4. Key design decisions

1. **Pivot from historical time series to present-day real-time.** The single most consequential decision in the project. Justified by perceptual coherence (map "now" + sound "now" beats map "now" + sound "past") and by relocating the sonification's job from numeric fidelity to *atmosphere/comparison*. Set the philosophical frame for all later sound-design work.
2. **Both modes coexist instead of replacing each other.** Per-grid mode does not suppress aggregated OSC. Lets Max choose; preserves the simpler aggregated patch as a fallback; means the threshold transition does not require Max to switch architectures, only to crossfade.
3. **Server-side spatial pre-computation** (`/grid/pos`). Rather than ship absolute lon/lat and expect Max to compute viewport-relative position, the server normalizes once per viewport update outside the per-cell loop. Cheaper, fewer arithmetic objects in the patch, and centralizes the cell-center offset (the `+ GRID_SIZE/2`) in one place.
4. **Additive OSC extension policy.** New addresses (`/mode`, `/grid/pos`, `/grid/lc`) added alongside existing ones rather than modifying. Established the project-wide rule that survived through M4's wire-format compatibility gate.
5. **Null-preserving normalization.** `landcover_class` empty/missing returns null, never defaulted to class 10 (Tree/Forest). Avoided silently inflating forest fractions — a small fix at M1 with outsized data-integrity payoff later.

## 5. Deferred / out-of-scope at M1 close

Not failures — explicit deferrals.

- **`poly~` voice allocation, spatial panning sound design, fade-in/out envelopes, mode-switch crossfade** — all on the Max side. M1 stopped at "data verification with `print` objects." Max sound design proper picks up in M2.
- **Threshold tuning** (PER_GRID_THRESHOLD = 50). Listed as an open question, never re-tuned during M1.
- **Mode-switching transition timing** (~500 ms crossfade was hypothesized; never measured).
- **Volume strategy for same-class grids** (minority prominence vs. equal-volume; needs listening tests).
- **Multichannel beyond stereo** (quad/ambisonics).
- **Acoustic ecology angle** (Nature's Soundscape, BC) — referenced once, not pursued.

## 6. What M1 set up for downstream milestones

- **M2** inherited the OSC contract and built sound design on top of it (crop bus, water bus, tree bus, the seven-bus ambience, server-side foundation refactor, eventual frontend module split).
- **M3** kept the wire format byte-identical when migrating from Max/MSP-via-OSC to the in-browser Web Audio engine — the `/lc/*` 11-float distribution, the `/mode` string, the `/grid/pos` normalization all carried forward as JSON message keys.
- **M4** merged the per-client state (WS-per-connection + HTTP-per-IP-with-TTL pattern from M1 §3.3) into the unified `client-state.js`.
