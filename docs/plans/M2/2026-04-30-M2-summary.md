## 1. Theme

M2 turned M1's data pipeline into **sound**. It also performed the single largest architectural pivot in the project's history: midway through M2, the Max/MSP-via-OSC sound engine was replaced wholesale by a **browser-native Web Audio engine**. By the close of M2, the OSC pipeline had been deleted from the codebase entirely.

The milestone has two clean halves separated by 2026-02-22:

- **Pre-Phase-W (2026-02-19 → 2026-02-20):** Max-side sound engine designed and built — `crossfade_controller.js`, `water_bus.js`, `icon_trigger.js`, `granulator.js`, `loop_clock.js`, `loop_voice.js`, `loop_bus.maxpat`. Server gains six new control signals (`/mode`, `/proximity`, `/delta/*`, `/coverage`). Bus topology evolves from 3 → 4 (Water bus split) → 5 (Crop bus extracted).
- **Post-Phase-W (2026-02-22 → 2026-03-16):** Max/MSP and OSC code deleted in commit `753b564`; `frontend/audio-engine.js` becomes the new sound engine. Subsequent work tunes the Web Audio loop crossfade, fixes phase drift, replaces threshold-based ocean detection with coverage-linear, retunes the low-pass filter, adds the loop-progress bar, splits the Tree bus into Forest/Shrub/Grass (5 → 7 buses).

The seven-bus ambience that ran through M3, M4, M5, and M6 was **finalized at M2 close** (commit `4384add`, devlog [`2026-03-16-M2-split-tree-bus.md`](../../devlog/M2/2026-03-16-M2-split-tree-bus.md)).

After `753b564` the project is **Web Audio only** — no `osc.js`, no `osc_schema.js` as an OSC carrier (the schema concept survived as the wire-format contract for WS messages until M4 §11), no Max patch in the runtime path. `frontend/audio-engine.js` is now the canonical sound engine and the file that grows into the 1186-LOC monolith broken up in M4.

The Phase W pivot is recorded here for the first time. Earlier diary-style entries (`Explain recent file changes`, `Debug loop playback issue`) document micro-progress but do not stand as a milestone-level rationale; in retrospect the pivot was driven by the same observation that motivated M3's deployment — Max/MSP cannot ship as a public website, and even pre-deployment the dual-stack (Max + browser map) was more friction than the project could justify for a single-author build.

## 3. Architecture at M2 close

### 3.1 Wire format (WS messages, post-Phase-W)

The OSC schema concepts (canonical class order, additive-only extension, ordering: mode → proximity → delta → lc → coverage) carried forward as the WebSocket message contract. Wire signals at M2 close:

- `mode` (string `"aggregated"` / `"per-grid"`)
- `proximity` (float 0–1, **zoom-derived**)
- `delta.lc` (11 floats, per-class change since previous frame)
- `delta.magnitude` (float 0–1)
- `delta.rate` (float 0–1)
- `coverage` (float 0–1, fraction of viewport with grid data)
- `lc[]` (11 floats, area fractions; classes not present send 0.0)
- `landcover` (int, dominant class) + `nightlight` / `population` / `forest` (float 0–1)
- Per-grid mode adds `grid[]` with `lon, lat, lc, nl, pop, forest`, `gridPos[]` with `xNorm, yNorm`, `gridLc[]` with 11 floats per cell

`audioParams.oceanLevel` was added then **removed** during M2 (last act of `2026-03-16-M2-doc-code-sync`); the frontend computes ocean mix from `coverage` directly.

## 4. Key design decisions

1. **Phase W pivot (Max/MSP → Web Audio).** The single most consequential M2 decision. Justification not formally written down but inferable: Max/MSP cannot ship as a public website (M3's eventual goal), and the dual-stack overhead (Max patch + JS map + OSC simulator + scenario tests) dominated build time for a single author. Web Audio loses some Max-specific tooling (`granulator.js`, `icon_trigger.js`, `loop_clock.js` Task scheduling) but gains a single-runtime stack, a shippable surface, and the same harmonic-coherence guarantees (global swap clock + A/B voice swap on the JS side mirrors `loop_clock.js` + `loop_voice.js` on the Max side).
2. **Ocean = absence of grid data, not a land-cover class.** Established at the Water bus introduction (2026-02-19) and survived every ocean-rule rewrite (3-level → piecewise → linear). Lets the system stay audible over open ocean where M1's `/lc/*` would all be 0.
3. **Coverage-linear over thresholded ocean detection.** Three rewrites in three days (3-level → piecewise 0/0.2/0.22 → linear 0/0.4) converged on the simplest rule with the smoothest behavior. Rule: `landMix = clamp(coverage / 0.4)`, `oceanMix = 1 − landMix`. Lasted unchanged through M3/M4/M5/M6.
4. **Power curve on bus gain (`^0.6`).** Stretches perceptual contrast in green-dominant viewports where the linear curve compresses everything to 0.85–1.0. Introduced alongside the 7-bus split.
5. **Global swap clock with A/B voices.** Same rationale on JS side as on Max side — harmonic content (chord-progression WAVs) cannot crossfade independently per bus or chords collide. Late-swap phase alignment (start incoming at offset, advance clock from planned not callback time) preserves the boundary across browser-tab background throttling.
6. **Removed `oceanLevel` from server payload.** Sending a server-side `oceanLevel` while the frontend computed its own from `coverage` was a misleading signal. The doc-code-sync pass (2026-03-16) deleted it and all golden-fixture references — set the precedent that the wire format should not carry derived signals the frontend can re-derive cheaply.
7. **Additive-only OSC/wire schema** carried forward from M1. Every new signal (`/proximity`, `/delta/*`, `/coverage`, `/grid/lc`, `/grid/pos`) added a new address; never modified existing payload shapes. M3 inherited this discipline; M4 codified it as a smoke gate.

## 5. Items that did not land in M2

- **`icon_trigger.js`** — code-complete on the Max side, never built on the Web Audio side. Surfaced again as the M3 city-name announcer (different mechanism but similar intent: short data-driven event triggers).
- **`granulator.js`** — code-complete on the Max side, never ported to Web Audio.
- **Per-grid spatial panning** — `/grid/pos` was sent (M1) and survived (M2 wire format), but no Web Audio voice allocator consumed it. Per-grid mode itself stayed on the wire as a `mode` flag without producing per-cell audio in the engine.
- **Latitude-driven `playbackRate` modulation** — added experimentally, removed in `2026-03-16-M2-loop-retrigger-phase-lock` because it broke the global swap clock. The `updateMotion()` lat argument is preserved for a future timing-safe modulation path.
- **Audio idle CPU measurement** — accepted as future work; eventually picked up in M4 P5-1 (with the rAF idle-suspension feature).
