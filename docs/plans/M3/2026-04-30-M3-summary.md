## 1. Theme

M3 is the milestone that **shipped the project to production** (`placeecho.com`) and reframed it from a Max/MSP-adjacent course project into a usable web instrument. The work has three concurrent threads:

1. **City-name announcer** — a new wayfinding layer on top of the seven-bus ambience.
2. **Production deployment** — Cloudflare Pages (frontend) + R2 (large assets) + Cloudflare Worker (subpath proxy) + Fly.io (Node.js backend) + Cloudflare auto-deploy + GitHub-Action-driven Fly deploy. Single-port server. Opus 20× audio compression. PMTiles asset hosting. `placeecho.com/geo-sonification/` live with full HTTPS, atomic Pages switchover, and zero-touch deploy.
3. **UX / audio polish for first audience** — info-panel visual overhaul, master loudness normalization (−16 LUFS), announcer sidechain ducking, accent unification, dot-overlay rendering iterations (six approaches tried, fifth reverted), drag-perf bundle (gzip + perMessageDeflate + cache headers + debounce + stroke suppression).

The branch ran from the deprecation of the Open Platform plan (~2026-02-23) through the merge of `feat/M3-ui-ux-overhaul` to `main` on 2026-04-25 via PR #5 (commit `f6ced39`). There is no explicit `chore: close milestone 3` marker commit (the pattern M4 later adopted with `a173ea4`); the milestone effectively ended when M4 began on 2026-04-27 with the razor-refactor proposal.

### 3.4 Audio chain at M3 close

```

  → masterGain (user volume slider, range 0–100)
```

## 4. Key design decisions

1. **Reframe from "Open Platform" to "UX/audio iteration + production deployment".** The deprecation of the open-platform proposals (`docs/plans/deprecated/2026-02-21-M3-open-platform-migration/`) was the single most consequential M3 decision and the one most poorly documented at the time — there is no "deprecation devlog." The branch name `feat/M3-ui-ux-overhaul` is the most explicit record of the new scope. Justification implicit: the open-platform spec assumed a multi-tenant API surface; the project at M2 close was a single-tenant interactive demo. Building an open-platform layer before the demo found its first audience would have been speculative scope. M3's actual scope (deployment + polish) is the minimum needed to reach a first audience.
2. **Single-port server** — kept dev = prod by listening on `$PORT`, attaching WS to the HTTP `upgrade` channel. Avoids per-environment branching in both `server/index.js` and the frontend URL builder.
3. **Reverse-proxy Worker over Pages custom domain.** Pages domains bind only at zone apex/subdomain, not path prefixes; reserving `placeecho.com` for an eventual main site forces `/geo-sonification/*` as a sub-path. ~5 ms p50 edge hop is cheap relative to the rest of the load.
4. **Static loudness calibration with peak limiter, not real-time LUFS-aware auto-gain.** Real-time BS.1770 is ~200 LOC of FFT + filter banks and produces audible pumping on ambient material. One-shot calibration + soft limiter is the right tool for ambient continuous content.
5. **Per-bus preamp instead of re-authoring WAVs.** Urban's ~15 dB peak imbalance is a source-authoring problem; addressed at runtime via `BUS_PREAMP_GAIN[urban] = 0.316` without touching the WAVs. Lets the master makeup go to +12 dB without limiter pumping on urban.
6. **Sidechain ducking via automation, not Worklet.** Web Audio `DynamicsCompressorNode` has no sidechain input; AudioWorklet would be the "real" path. For single-voice / single-event speech, `setTargetAtTime` on `duckGain` gives the same perceptual result with far less plumbing.
7. **Reverted six dot-rendering experiments to pre-LOD baseline.** All six approaches (per-feature minzoom / cluster-distance / drop-fraction-as-needed / two-tier square / sub-pixel radius / radius bump) had trade-offs the user judged not worth shipping. Documenting six rejected approaches *with their reasoning* is a positive record for the next time someone touches this problem — the rejected commits stay in git history and devlog index.
8. **Drag-state ≠ rest-state**. The "no static-state LOD" veto applies to rest-state appearance; drag-state can drop sub-pixel detail (motion blur masks it) → stroke suppression on `movestart` halves fragment shader cost without perceptible loss.
9. **Stale warning by grace window, not backoff proxy.** `wsReconnectDelay >= 4000` was a leaky abstraction that fired on backoff doubling, not actual data age. 5 s grace timer started on disconnect, cancelled on reconnect, is the correct semantic.
10. **Two-tier brand (PlaceEcho parent, Geo-Sonification service).** Driven by USPTO Section 1(a) specimen requirements (TMEP §1301.04(f)(ii)). Footer-only branding fails the standard; first attempt as `<h2>PlaceEcho</h2>` + subtitle `Geo-Sonification` failed editorial intent.
